C / C++

The C Programming Language, designed by Dennis Richie in 1973. It's an imperative language, designed for use close to the hardware level. Still supported and developed to this day. Primarily standardized by as ANSI C.

According to the creator of C++, Bjarne Stroustrup, C++ was always meant to be an expansion to C, and not a separate language. As it is, C++ is a object oriented compiled language that works real fast and introduces a significant amount of very useful native functionality that significantly modernizes the C language. A great deal of overlap exist, where considerable support for C programs, syntax and resources exist directly in the language itself.

BASIC FILE STRUCTURE

This section clarifies the very basics of how a program file needs to be organized in c/c++ for the compiler to successfully be able to parse it.

Conventionally, the file ending of a c file is .c, and a c++ file is .cpp.

C LANG FILE EXAMPLE

#include <header-file.h> #include 'library-file.c' /* declare functions before defining them up here */ int main(int argc, char const *argv[]){ //Main program code goes here return some_int; }

C++ FILE EXAMPLE

#include <cstdlib> #include <iostream> using namespace std; int main(int argc, char const *argv[]){ /*Program content*/ return some_int; }

DIRECTIVES

Preprocessor commands are supplied at the top of the document and must be prepended with a # character with no whitespace between it and the line start.

FILE INCLUSION

The core syntax offer only the very basic of primitive datatypes and flow control. Any level of advanced support requires the inlusion of libraries in one way or another.

The #include statement added at the top of the file signifies which resources are to be included. #include <stdio.h> will include the standard input output library. #include "some/file.h" will instead include any arbitrary file with the path offered.

Once a file has been included, any function or variable in its global scope will be available to the including file.

DEFINITIONS

Constant value can be defined for practical use later in a global scope. Use the define directive like so: #define CONSTANT value. These defenitions will always be constant, they can't be written in runtime. They don't have to be named with capital letters, but the convention is such.

COMPILING

In a linux environment, a standard compiler, for c and c++ both, is the Gnu C Compiler gcc, easily invoked with:

gcc path/to/program.c -o path/to/executable
Note that if the input does not have a file ending, the language must be specified with the -x option.

c++ programs are compiled in a similar fashion, but requires an explicit linking instruction to access the c++ libraries. This is achieved with the -lstdc++ flag.

gcc path/to/program.cpp -lstdc++ -o path/to/executable

GENERICS, TEMPLATES & INTERFACES

Generic programming is the concept of designing functions and classes that are not dependent on a specific datatype. Important to make code properly modular and generally applicable.

FLOATING POINTS

Floating Points are a formulaic representation of approximate fractions. The implementation of floating points give rise to a level of errors that which mean that two operations that are mathematically equivalent, might get different results in practice.

The datatypes float or double , FP32 and FP64 (bits) respectively, are both actually floating points, but with differing precision. Common terms are half-precision, singe precision and double precision.

A general overview is found on the wikipedia page for floating point arithmetic. Details on comparing from a programmers perspective is discussed on floating-point-gui.de and randomascii. The, most common, standard is the IEEE 754. A very practical graphical illustration of how the bit representation becomes a real number can be found on this github pages site by Evan Wallace.
