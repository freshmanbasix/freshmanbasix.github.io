The Key-Pair Model

Of all available encryption methods, one very common and reliable kind is the private public keypair model. The idea is that a user has two keys; one private that can decrypt (unlock) messages meant only for the owner, and one public that encrypts (locks) a message meant for a specific recipient.

Gnu Privacy Guard

GPG is a open source and readily available command line utility that enables a large amount of functionality and management. This paragraph will expand on the interesting history of GPG, it's quite a doozie

Usage

Below follows a number of examples of how one might use the gpg command achieve some serious privacy. First you will want to encrypt a input-file intended to be read only by the designated recipient UIDs.

$ gpg --encrypt <input-file> --armor --recipient UID --output <output-file>

When you've received an encrypted-message, you're going to want to decrypt it and output the unencrypted message to an appropriate output-file.

$ gpg --decrypt --output <output-file> <encrypted-message>

In order to share your own key with other people, export it into a file that can be sent through any prefered means. 

$ gpg --export <uid> --armor --output <filename>

Keep in mind that reliable trust is an essential part of the system. Make absolutely certain that the key you have received is the correct one. The simplest way is to compare fingerprints. To see an overview of all keys in your keyring and the respective UIDs and fingerprints, use the --list-keys command.

$ gpg --list-public-keys
