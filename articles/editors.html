When all other wars have been fought and lost, there is always the fight over which editor is better. The answer is of course whichever one you like. Just be sure to try a few different ones.

In this section, a few of them will be listed and primarily quick links to useful resources will referenced. To some extent, this article might contain some instructions on customizing the editor itself.

ATOM

Find the official manual in this Atom Flight Manual.

Create custom macro snippets by adding them line by line in ~/.atom/snippets.cson. Be sure to take care of the indentation and follow the general format. Read more in the atom snippet documentation

'.text.language-package-snippet-applies-to': 'first snippet name': 'prefix': 'snippet macro shortcut' 'body': 'content' 'second sippet name': 'prefix': 'snippet macro shortcut' 'body': """ multi line content tab to pointer at $1 first. For prefilled content, use ${2:this format} """

LATEX

Formally a layout language, rather than an editor. The primary place to use latex for making documents is the overleaf web editor.

practical resources for latex

Latex Project main site https://www.latex-project.org/help/documentation/#general-documentation

OVERLEAF

https://www.overleaf.com/ is an excellent browser based latex editor that simplifies the process of generating latex documents. It does not have a complete, but more than sufficient customizability in terms of package inclusion and compilation specifications. A further benefit is enabling real time collaboration and has a fairly decent privacy policy.

LianTze Lim has made a nice cheatsheet for keyboard shortcuts that make writing quicker. One might also have a look at the guide and tutorial overview right here.
