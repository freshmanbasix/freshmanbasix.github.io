In this article, complete instructions on how to practically go about making a contribution to the wiki will be documented. For the foreseeable future, it will be required by an author to get aquainted with the particular way that the website is designed and styled. Be aware that these particular solutions might not always be optimal, or even very good, from a conventional perspective. This does leave a great deal of room to make contributions that improve on these rag-tag, homemade design decitions. If you find something on this wiki that is crap and you have a good idea on how it might become less so, then keep reading this article and submit your suggestions.

External Resources

The wiki is compeletely stored on a Git repository, meaning any change is done through pull requests. This offers an existing open source framework for managing, moderating and administrating the wiki without any need for user accounts. The repo is currently publicly hosted on Github and you'll find it right here.

For managing active maintenance and tasks pertaining to the wiki, there's a publicly readable Trello board which can be used by anyone to view what sort of improvements have already been identified and what is being worked on.

If you would like to grab an improvement task for yourself, or perhaps suggest one, an email is the way to go. Just shoot an email to hjalle@freshmanbasix.com. If interest ever grows large enough, there might appear a discord server as well. Time will tell.

HTML Map Overview

The core content of the wiki is contained within a CSS flexbox dividing itself into 3 columns where the central columns contains the main article and is 4 times as wide as the outer ones.

Navigation Bar

The leftmost column offers an overview and navigation of articles. Under the nav element with id="nav-bar" is an unordered list, with each li formated as follows:

<li class="nav-item">
  <a href="javascript:openArticle('article-file-name')">Title</a>
</li>

Each list-item must have the class attribute set to nav-item and contain an anchor element with the href attribute set to call the javascript function openArticle().

Wiki Article Column

The article itself is contained within the main element with id wiki-column. The whole article itself is stored inside a article element which has an id value corresponding to the article file name, as found in the /articles directory. It also has the class value wiki-entry. In general, the article is divided into several section elements, the first of which also contains the table of contents. See a core example code below.

<article id="file-name" class="wiki-entry">

  <h1 class="column-header">Title</h1>

  <section class="sub-section">

    <div id="toc-container">
      <h1 class="column-header toc-header">Table Of Contents</h1>
      <ul class="toc">
        <li><a href="#sub-section-1">Sub Header</a></li>
      </ul>
    </div>

    <p>Introduction</p>

  </section>

  <h2 id="#sub-section-1">Sub Header</h2>

  <section class="sub-section">

    <p> Sub section content </p>

  </section>

</article>

Note that each h2 header preceeds the section element for each sub-section. This was chosen because it made it simpler to reliably make the margins larger for the text content than the header itself. It does feel slightly unintuitive though, so an alternate solution is being worked on.

Formating and style-rules

The styling in the wiki is built on the assumption that all articles are authored and formated in a consistent way. There are a number of predefined blocks that can be used to present information in a effective way. Below follows the available elements that can be used, along with necessary attributes.

Some thought on what to aim for and how to overall structure content will be added here.

  1. Sub-sub-headers. Two levels of sub-sub-sections are supported by the css styling. h3 and h4 elements are all that is needed.
  2. Paragraphs. The plain <p> element is all that is required. Each paragraph of the content need an element on its own.
  3. Figures. Images can be inserted using the <figure> element along with the <img> and <figcaption> elements. 
    <figure>
  <img src="filepath" alt="">
  <figcaption>Caption</figcaption>
</figure>

    Take note that for any diagram the caption should include a link to a xml file that can be used to edit the image itself without startng from scratch. Source files are all stored in the /rsc directory.

  4. Code snippets. A <pre> element with the class attribute set to "box snippet" will render a neat little code example box. 
    <pre class="box snippet">Write code here</pre>

    Be aware that the pre element renders text as written in the html file, which includes the indentation. So if you don't push it back to the beginning of the line manually, it's going to look bonkers.

  5. Attention block. A good way to make some important information stick out like a smack in the face is the attention block. It is achieved by adding a class value to a p element.

    This is an example of the attention box

    <p class="box attention"> Write important info here</p>
