HyperText Markup Language

I have no idea what's so damn hyper about it, but the language is a syntax for marking up data to enable it's presentation to be rendered by some other software. The complete language set is very small, consisting of a few central components making up the syntax glossary. There's just a few of them, illustrating how quick HTML truly is to review and start using

Syntax Glossary

  1. First out is the HTML ELEMENT, exemplified below with a <img> element. This is the core building block if HTML, consisting of two TAGS, much like a set of parenthesees, who OPEN and CLOSE the element. While some circumstances allow for the two to be combined, or the closing tag to be omitted, using both is the only certain way to avoid undefined behavior. 
    <img src="file"> </img>
  2. Secondly comes the ATTRIBUTE. These are descriptors defining the behavior and presentation of the Element. Attributes are always located inside the opening tag and separated by white spaces. Generally speaking, most attributes use in practice are specific for the element. Styling is possible through attributes, many CSS properties have attribute equivalents, but it's a BAD IDEA! 
    src="file"
  3. Any element will have ONE PARENT and ANY NUMBER of CHILDREN, i.e one element of which said element is inside and zero or more elements in between the tags. 
    <figure>
  <img src="/img/file.png" alt="text"></img>
  <figcaption>Here's a caption</figcaption>
</figure>
